การ copy data จาก service ต้นทาง มาไว้ที่ db กลาง และให้ service ปลายทางมาดึงข้อมูลไป

	เช่น
		เมื่อมีการสร้าง account ที่ account service ก้จะให้ copy data ไปที่ db กลางด้วย (publish db)

		เมื่อเมื่อมี service ไหนต้องการ ก็ให้เข้ามาดึงข้อมูลได้ เช่น report service ต้องการข้อมูล


		โดยสิ่้งที่ทำให้ publish db คือ ทางข้อมูลต้นทางอาจมีข้อมูลจำนวนมาก(หลาย table) ก็ให้ทำ denormalization (รวมข้อมูล) ไว้ที่ db กลาง
		เพื่อให้ service ปลายทางดึงข้อมูลไปใช้ง่ายขึ้น


มีอีกรุปแบบนึงคือ แทนที่จะเก้บข้อมูลไว้ที่ db กลาง , เราใช้ event base เข้ามาช่วย

สรุปคือเทคนิค storing copy of data สามารถใช้ได้ทั้ง db กลาง หรือ event base ก้ได้


ปัญหาที่ตามมา
	- กรณีมี db กลาง => มีค่าใช้จ่ายเพิ่ม เพิ่มความซับซ้อน
	-ต้นทาง ปลายทาง จะมีการผูกมัดมากขึ้น เพราะเรื่องของ format/รูปแบบของ event ต้องมีข้อตกลงร่วมกัน จึงไม่เป้นอิสระกันมากนัก
