จากนิยาม micro service จะแยก data ของแต่ละ service ออกจากกัน และแต่ละ db อาจมีข้อมูลที่สัมพันธ์กัน หรือซ้ำซ้อน และ service ต้องดึงข้อมูลมาทำงานร่วมกัน

	เช่น การโอนเงิน

	ผู้ใช้ต้องการโอนเงิน -> ถอนเงินจากบัญชีต้นทาง -> ฝากเงินเข้าบัญชีปลายทาง -> บันทึกการทำธุรกรรม
			|		   |	             |
			db		   db		     db

	ซึ่งแต่ละขั้นตอนอยู่คนละ service และแต่ละ service มีการจัดการ Tx ...แล้วทั้ง flow ของการทำงาน เราจะจัดการอย่างไร ??? เนื่องจากเราไม่สามารถจัดการ Tx ข้าม db ได้
	^
                     นี่ก็คือปัญหาอย่างนึงในการแยก service และถ้าเกิดปัญหาจะแก้อย่างไรในระหว่าง service => เช่นถอนเงินต้นทางได้ แต้ไปพังที่ฝากเงิน จะทำอย่างไร ??

	ซึ่ง microservice ทำการ roll back ไม่ได้ เพราะข้อมูลอยู่แยก db
